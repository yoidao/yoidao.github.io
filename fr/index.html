Codes d'erreur de l'API Youdao Zhiyun / diagnostic de l'API de traduction

L'interface de traduction / dictionnaire / OCR / TTS de Youdao Zhiyun a renvoyé un errorCode ? Clarifiez trois points d'abord : est-ce un problème de paramètre/signature vérifiable côté client, ou serveur/facturation, l'ordre et l'encodage de la chaîne de signature sont-ils corrects, et le QPS est-il dépassé. Saisissez le code d'erreur ou un mot-clé (ex. 202, 411, 108, signature, limite, horodatage) : d'abord la signification officielle L1, puis la cause racine et le diagnostic L2, enfin la comparaison avec les concurrents et des cas réels. Doc officielle sur le guide d'intégration / codes d'erreur Youdao Zhiyun.

Étape 1 : classer par plage de code (L1 - d'abord à qui la faute)

Le errorCode de Youdao Zhiyun se divise en quatre plages ; situez avant d'essayer à l'aveugle : ① 1xx paramètres (101 manquant / 102 langue / 103 long / 113 q vide / 116 strict) - le client vérifie les obligatoires et les valeurs ; ② 2xx auth & signature (108 ID / 202 signature / 203 IP / 205 plateforme / 206 horodatage / 207 rejeu) - vérifier la chaîne de signature et les identifiants ; ③ 3xx/5xx serveur (303 / 500 / 902000) - réessayer avec request_id ou ticket ; ④ 4xx compte/facturation/limite (401 solde / 411 / 412) - vérifier solde et QPS. Plus de détail sur la génération de signature v3 et le diagnostic du 202.

Étape 2 : les deux pièges les plus fréquents → cause racine (L2 : signature 202 et limite 411)

Essentiel : 90 % des erreurs de l'API Youdao se bloquent sur « signature 202 » et « limite 411 ». Le 202 est rarement une clé erronée, mais souvent une chaîne de signature v3 mal construite - sign = sha256(appKey + input + salt + curtime + appSecret), où input pour q>20 prend « 10 premiers caractères + longueur de q + 10 derniers caractères » (en caractères Unicode, pas en octets), et q si ≤20. Ajoutez l'encodage UTF-8, l'ordre et les espaces. Le 411 est un QPS au-dessus du forfait : limitez le débit + backoff côté client. Diagnostic rapide par scénario réel

  • OK en local, mais beaucoup de 206 en cloud/conteneur : l'horloge du conteneur dérive et invalide curtime (signature ~120s) → activez NTP, synchronisez l'horloge sur l'hôte ; ne cachez pas d'anciens curtime.
  • L'anglais se traduit bien, le chinois long donne parfois 202 : la troncature a coupé le multioctet par « octets » → input doit compter les caractères Unicode (q>20 : 10 + longueur + 10).
  • salt fixe ou auto-incrémenté et un 207 au second appel : le salt doit être aléatoire (UUID) à chaque fois et rafraîchir curtime ; au retry, re-signer.
  • Pics de concurrence en test de charge → beaucoup de 411 : QPS de base ~100 dépassé → token bucket + backoff côté client, ou acheter du QPS en console.
  • Nouvelle app appelant la prononciation et obtenant 110 : l'instance de traduction est liée, pas celle de TTS → liez chaque instance par service en console.
  • Soudain tout en 401 : pack de caractères / solde épuisé → rechargez et configurez des alertes de solde ; prévoyez un canal de secours sur les chemins critiques.
  • Même piège en migrant depuis Baidu : q encodé en URL avant la signature → Youdao v3 signe avec input (règle de troncature), pas avec q encodé ; signez d'abord, encodez q à l'envoi.
