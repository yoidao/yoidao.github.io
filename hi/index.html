Youdao Zhiyun API त्रुटि कोड / अनुवाद API निदान

Youdao Zhiyun के अनुवाद / शब्दकोश / OCR / TTS इंटरफ़ेस ने errorCode लौटाया? पहले तीन बातें स्पष्ट करें: क्या यह क्लाइंट द्वारा जाँचने योग्य पैरामीटर/हस्ताक्षर समस्या है, या सर्वर/बिलिंग की, क्या हस्ताक्षर स्ट्रिंग का क्रम और एन्कोडिंग सही है, और क्या QPS पार हुआ। त्रुटि कोड या कीवर्ड दर्ज करें (जैसे 202, 411, 108, हस्ताक्षर, सीमा, टाइमस्टैम्प): पहले L1 आधिकारिक अर्थ, फिर L2 मूल कारण और निदान, अंत में प्रतिस्पर्धी तुलना और वास्तविक मामले। आधिकारिक दस्तावेज़ Youdao Zhiyun API एकीकरण गाइड / त्रुटि कोड पर।

चरण 1: कोड परास के अनुसार वर्गीकृत करें (L1 - पहले किसकी गलती)

Youdao Zhiyun के errorCode मोटे तौर पर चार परास में बँटते हैं; अंधाधुंध आज़माने से पहले स्थान तय करें: ① 1xx पैरामीटर (101 गायब / 102 भाषा / 103 लंबा / 113 q खाली / 116 strict) - क्लाइंट अनिवार्य व मान जाँचे; ② 2xx प्रमाणीकरण व हस्ताक्षर (108 ID / 202 हस्ताक्षर / 203 IP / 205 प्लेटफ़ॉर्म / 206 टाइमस्टैम्प / 207 रीप्ले) - हस्ताक्षर स्ट्रिंग व क्रेडेंशियल जाँचें; ③ 3xx/5xx सर्वर (303 / 500 / 902000) - request_id के साथ पुनः प्रयास या टिकट; ④ 4xx खाता/बिलिंग/सीमा (401 शेष / 411 / 412) - शेष व QPS जाँचें। अधिक v3 हस्ताक्षर निर्माण और 202 निदान पर।

चरण 2: दो सबसे आम भूलें → मूल कारण (L2: हस्ताक्षर 202 और सीमा 411)

मुख्य बात: Youdao API की 90% त्रुटियाँ «हस्ताक्षर 202» और «सीमा 411» पर अटकती हैं। 202 शायद ही गलत कुंजी हो; प्रायः गलत बनी v3 हस्ताक्षर स्ट्रिंग - sign = sha256(appKey + input + salt + curtime + appSecret), जहाँ q>20 पर input «पहले 10 अक्षर + q की लंबाई + अंतिम 10 अक्षर» लेता है (बाइट नहीं, Unicode अक्षर के अनुसार), और ≤20 पर q। इसमें UTF-8 एन्कोडिंग, क्रम और रिक्त स्थान जुड़ते हैं। 411 योजना से अधिक QPS है: क्लाइंट पर दर सीमित करें + बैकऑफ। नीचे स्व-जाँच पैनल, फिर तुलना और 2026 अनुमान।

अनुवाद API त्रुटि प्रबंधन और कोटा (2026 अनुमान)

प्लेटफ़ॉर्मों के आधिकारिक दस्तावेज़ों (Youdao Zhiyun, Baidu, Tencent Cloud, Alibaba Cloud, Google, DeepL) से 2026 अनुमान: वचनबद्ध या प्रत्यक्ष मापे मान नहीं, केवल संदर्भ; कोटा व QPS आधिकारिक नीति से बदलते हैं:

अनुमान आधार: संदर्भ स्रोत + कालिक एक्सट्रापोलेशन (Youdao / Baidu / Tencent / Alibaba / Google / DeepL के 2026 आधिकारिक दस्तावेज़); नीति व संस्करण से बदलता; नवीनतम आधिकारिक कंसोल मान्य।

वास्तविक परिदृश्य से त्वरित निदान

  • लोकल पर चलता है, पर क्लाउड/कंटेनर पर बहुत 206: कंटेनर घड़ी खिसकने से curtime अमान्य (हस्ताक्षर ~120s) → NTP सक्षम करें, घड़ी होस्ट से सिंक; पुराना curtime कैश न करें।
  • अंग्रेज़ी ठीक अनुवाद, लंबी चीनी कभी-कभी 202: truncation ने मल्टीबाइट को «बाइट» से काटा → input को Unicode अक्षर गिनने चाहिए (q>20: 10 + लंबाई + 10)।
  • स्थिर या बढ़ता salt और दूसरे कॉल पर 207: salt हर बार यादृच्छिक (UUID) हो और curtime ताज़ा करें; retry पर पुनः हस्ताक्षर करें।
  • लोड टेस्ट में समवर्ती शिखर → बहुत 411: आधार QPS ~100 पार → क्लाइंट पर token bucket + बैकऑफ, या कंसोल से QPS खरीदें।
  • नया ऐप उच्चारण कॉल कर 110 पाता है: अनुवाद इंस्टेंस बाउंड है, TTS नहीं → कंसोल में प्रति सेवा प्रत्येक इंस्टेंस बाँधें।
  • अचानक सब 401: वर्ण पैकेज / शेष समाप्त → रिचार्ज करें और शेष अलर्ट सेट करें; महत्वपूर्ण पथों पर बैकअप चैनल रखें।
  • Baidu से माइग्रेशन में वही भूल: हस्ताक्षर से पहले q को URL-encode किया → Youdao v3 input से हस्ताक्षर करता है (काट नियम), एन्कोडेड q से नहीं; पहले हस्ताक्षर, भेजते समय q एन्कोड।
