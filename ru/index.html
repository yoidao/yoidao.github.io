Коды ошибок API Youdao Zhiyun / диагностика API перевода

Интерфейс перевода / словаря / OCR / TTS Youdao Zhiyun вернул errorCode? Сначала проясните три вещи: это проверяемая на клиенте проблема параметра/подписи или сервера/биллинга, верны ли порядок и кодировка строки подписи и не превышен ли QPS. Введите код ошибки или ключевое слово (напр. 202, 411, 108, подпись, лимит, метка времени): сначала официальное значение L1, затем корневая причина и диагностика L2, в конце сравнение с конкурентами и реальные случаи. Официальная документация на руководстве по интеграции / кодам ошибок Youdao Zhiyun.

Шаг 1: классифицируйте по диапазону кода (L1 - сначала чья вина)

errorCode Youdao Zhiyun делится примерно на четыре диапазона; локализуйте, прежде чем пробовать вслепую: ① 1xx параметры (101 нет / 102 язык / 103 длинно / 113 q пусто / 116 strict) - клиент проверяет обязательные и значения; ② 2xx аутентификация и подпись (108 ID / 202 подпись / 203 IP / 205 платформа / 206 метка времени / 207 повтор) - проверьте строку подписи и учётные данные; ③ 3xx/5xx сервер (303 / 500 / 902000) - повторите с request_id или тикет; ④ 4xx аккаунт/биллинг/лимит (401 баланс / 411 / 412) - проверьте баланс и QPS. Подробнее на генерации подписи v3 и диагностике 202.

Параметры101 · 102 · 103 · 113 · 116
Проверка клиента: обязательные поля полны (q/from/to/appKey/sign/signType/curtime/salt), коды языка zh-CHS / en / auto, q не пусто/не слишком длинно, strict true/false.
Аутентификация / подпись108 · 111 · 202 · 203 · 205 · 206 · 207
SHA256 от appKey+input+salt+curtime+appSecret; curtime UTC в секундах, salt UUID против повторов; белый список IP согласован с типом платформы.
Сервис / экземпляр110 · 310
Каждое приложение привязывает экземпляр на сервис (перевод / TTS / OCR отдельно); доменный перевод включается заранее в консоли.
Аккаунт / биллинг / лимит401 · 411 · 412
Не исчерпывайте баланс / пакет символов (401 останавливает сервис); QPS (базовый ~100) превышен = 411: token bucket + экспоненциальный backoff.
Результат / расшифровка / прочее201 · 303 · 500 · 902000
Ошибки сервера: повторите с request_id или тикет; в шифрованном интерфейсе проверьте порядок DES / BASE64 / URLDecode.

Шаг 2: две самые частые ловушки → корневая причина (L2: подпись 202 и лимит 411)

Главное: 90% ошибок API Youdao застревают на «подписи 202» и «лимите 411». 202 редко из-за неверного ключа, обычно это неправильно собранная строка подписи v3 - sign = sha256(appKey + input + salt + curtime + appSecret), где input при q>20 берёт «первые 10 символов + длину q + последние 10 символов» (по символам Unicode, не байтам), а q при ≤20. Добавьте кодировку UTF-8, порядок и пробелы. 411 - это QPS выше тарифа: ограничьте частоту + backoff на клиенте. Ниже панель самопроверки, затем сравнение и оценки на 2026.

Быстрая диагностика по реальному сценарию

  • Локально работает, но много 206 в облаке/контейнере: часы контейнера дрейфуют и обнуляют curtime (подпись ~120s) → включите NTP, синхронизируйте часы с хостом; не кэшируйте старые curtime.
  • Английский переводит хорошо, длинный китайский иногда 202: усечение порезало многобайтовое по «байтам» → input должен считать символы Unicode (q>20: 10 + длина + 10).
  • фиксированный или растущий salt и 207 на втором вызове: salt должен быть случайным (UUID) каждый раз и обновлять curtime; при повторе подпишите заново.
  • Пики параллелизма в нагрузочном тесте → много 411: базовый QPS ~100 превышен → token bucket + backoff на клиенте, или докупите QPS в консоли.
  • Новое приложение вызывает произношение и получает 110: экземпляр перевода привязан, TTS - нет → привяжите каждый экземпляр на сервис в консоли.
  • Внезапно всё 401: пакет символов / баланс исчерпан → пополните и настройте оповещения о балансе; держите резервный канал на критичных путях.
  • Та же ловушка при миграции с Baidu: q URL-кодирован до подписи → Youdao v3 подписывает с input (правило усечения), не с кодированным q; сначала подпишите, кодируйте q при отправке.
