有道智云 API エラーコード / 翻訳エラー検索

有道智云の翻訳 / 辞書 / OCR / TTS 呼び出しで errorCode が返った？まず 3 つを固めます——クライアント側のパラメータ / 署名の問題か、サーバ / 課金の問題か、署名文字列の順序とエンコードは正しいか、QPS を超えていないか。受け取ったコードまたはキーワード（例：202、411、108、署名、レート制限、タイムスタンプ）を検索し、まず L1 公式の意味、次に L2 根本原因と対処、最後に競合比較と実例を確認。公式の接続ドキュメントは 有道智云 API エラーコード / 接続ガイド へ。

ステップ1：エラーコードの帯で切り分け（L1・誰のせいか）

有道智云の errorCode は概ね 4 帯。まず帯で位置づけ、当てずっぽうにしない：① 1xx パラメータ（101 欠落 / 102 言語 / 103 長すぎ / 113 q 空 / 116 strict）——クライアント側で必須と値を確認；② 2xx 認証・署名（108 アプリID / 202 署名 / 203 IP / 205 プラットフォーム / 206 タイムスタンプ / 207 リプレイ）——署名文字列と資格情報を確認；③ 3xx/5xx サーバ側（303 / 500 / 902000）——request_id 付きで再試行かチケット；④ 4xx アカウント/課金/レート制限（401 残高不足 / 411 / 412）——残高と QPS を確認。詳しい接続比較は 有道 API 署名(v3)生成と 202 のトラブルシュート。

パラメータ101 . 102 . 103 . 113 . 116
クライアント側で確認：必須項目が揃っているか（q/from/to/appKey/sign/signType/curtime/salt）、言語コードは zh-CHS / en / auto、q は空でなく長さ内、strict は true/false。
認証 / 署名108 . 111 . 202 . 203 . 205 . 206 . 207
署名文字列 appKey+input+salt+curtime+appSecret を SHA256；curtime は秒単位 UTC、salt は UUID でリプレイ防止；IP 許可リストとアプリのプラットフォーム種別を一致させる。
サービス / インスタンス110 . 310
1 つのアプリは使う各サービスのインスタンスを個別に紐付け（翻訳 / 音声合成 TTS / OCR は別々）；領域翻訳はコンソールで先に有効化。
アカウント / 課金 / レート制限401 . 411 . 412
残高 / 文字パックを切らさない（401 は残高不足で停止）；QPS（基本約 100）超過で 411——トークンバケットのレート制限 + 指数バックオフを。
結果 / 復号 / その他201 . 303 . 500 . 902000
サーバ側の異常は request_id 付きで再試行かチケット；暗号化エンドポイントは DES / BASE64 / URLDecode の順序を確認。

ステップ2：最頻の二大落とし穴 → 根本原因（L2・署名 202 とレート制限 411）

要点：有道 API エラーの 9 割は「202 署名」と「411 レート制限」で詰まる。202 は鍵の誤りより、v3 署名文字列の組み立てミスが大半——sign = sha256(appKey + input + salt + curtime + appSecret)。input は q が 20 文字超なら「先頭 10 文字 + q の長さ + 末尾 10 文字」（バイトでなく Unicode 文字数で数える）、20 以下なら q そのもの。さらに UTF-8 エンコード・順序・前後の空白の罠が重なる。411 はプランの QPS 超過——クライアント側でレート制限とバックオフを。下図は呼び出し前のセルフチェック盤。さらに下に競合比較と 2026 推計。

実例・クイック判定

  • ローカルは通る、クラウド / コンテナ移行後に大量 206：コンテナの時計ずれで curtime が無効（署名は約 120 秒有効）→ NTP で時刻同期、コンテナをホスト時計に同期、古い curtime をキャッシュしない。
  • 英語は正常、長い中国語で時々 202：truncate がマルチバイト中国語を「バイト」で切っている → input は Unicode 文字数で数える（q>20 は先頭 10 文字 + q の長さ + 末尾 10 文字）。
  • salt を固定 / 連番にして 2 回目で 207：salt は毎回ランダム（UUID）にして curtime を更新；再試行は必ず再署名し、そのまま再送しない。
  • 負荷試験で大量 411：基本 QPS は約 100、超えると制限 → クライアントでトークンバケットのレート制限 + 指数バックオフ、またはコンソールで QPS を追加購入。
  • 新規アプリで翻訳の発音を取って 110：翻訳インスタンスは紐付け済みだが音声合成（TTS）インスタンス未紐付け → コンソールでサービスごとに紐付け。
  • 急に全部 401：文字パック / 残高が尽きて停止 → チャージかパック購入と残高アラート設定、重要経路には代替チャネルを。
  • 百度から持ち込んだ同じ罠：署名作成前に q を URL エンコード → 有道 v3 は input（切り出し規則）を署名に使う、先に q をエンコードしない；先に署名、送信時に q を URL エンコード。
