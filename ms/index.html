Kod ralat API Youdao Zhiyun / diagnosis API terjemahan

Antara muka terjemahan / kamus / OCR / TTS Youdao Zhiyun mengembalikan errorCode? Jelaskan tiga perkara dahulu: adakah masalah parameter/tandatangan yang boleh disemak klien, atau pelayan/pengebilan, adakah susunan dan pengekodan rentetan tandatangan betul, dan adakah QPS melebihi had. Masukkan kod ralat atau kata kunci (cth. 202, 411, 108, tandatangan, had, timestamp): dahulu maksud rasmi L1, kemudian punca akar dan diagnosis L2, akhir sekali perbandingan pesaing dan kes sebenar. Dok rasmi di panduan integrasi / kod ralat Youdao Zhiyun.

Langkah 1: kelaskan mengikut julat kod (L1 - dahulu salah siapa)

errorCode Youdao Zhiyun terbahagi kira-kira empat julat; letakkan dahulu sebelum cuba membabi buta: ① 1xx parameter (101 hilang / 102 bahasa / 103 panjang / 113 q kosong / 116 strict) - klien semak wajib dan nilai; ② 2xx auth & tandatangan (108 ID / 202 tandatangan / 203 IP / 205 platform / 206 timestamp / 207 main semula) - semak rentetan tandatangan dan kelayakan; ③ 3xx/5xx pelayan (303 / 500 / 902000) - cuba semula dengan request_id atau tiket; ④ 4xx akaun/pengebilan/had (401 baki / 411 / 412) - semak baki dan QPS. Lagi di penjanaan tandatangan v3 dan diagnosis 202.

Parameter101 · 102 · 103 · 113 · 116
Semak klien: medan wajib lengkap (q/from/to/appKey/sign/signType/curtime/salt), kod bahasa zh-CHS / en / auto, q tidak kosong/terlalu panjang, strict true/false.
Auth / tandatangan108 · 111 · 202 · 203 · 205 · 206 · 207
SHA256 bagi appKey+input+salt+curtime+appSecret; curtime UTC saat, salt UUID anti-main-semula; senarai putih IP selaras jenis platform.
Perkhidmatan / instance110 · 310
Setiap app ikat satu instance per perkhidmatan (terjemahan / TTS / OCR berasingan); terjemahan domain diaktifkan dahulu di konsol.
Akaun / pengebilan / had401 · 411 · 412
Jangan habiskan baki / pakej aksara (401 hentikan perkhidmatan); QPS (asas ~100) melebihi = 411: token bucket + backoff eksponen.
Hasil / nyahsulit / lain201 · 303 · 500 · 902000
Ralat pelayan: cuba semula dengan request_id atau tiket; pada antara muka tersulit semak susunan DES / BASE64 / URLDecode.

Langkah 2: dua perangkap paling kerap → punca akar (L2: tandatangan 202 dan had 411)

Inti: 90% ralat API Youdao tersangkut pada «tandatangan 202» dan «had 411». 202 jarang kerana kunci salah, biasanya rentetan tandatangan v3 tersusun salah - sign = sha256(appKey + input + salt + curtime + appSecret), di mana input apabila q>20 ambil «10 aksara pertama + panjang q + 10 aksara terakhir» (mengikut aksara Unicode, bukan bait), dan q jika ≤20. Tambah pengekodan UTF-8, susunan dan ruang. 411 ialah QPS melebihi pelan: hadkan kadar + backoff di klien. Di bawah panel semak sendiri, kemudian perbandingan dan anggaran 2026.

Pengendalian ralat dan kuota API terjemahan (anggaran 2026)

Anggaran 2026 daripada dokumentasi rasmi setiap platform (Youdao Zhiyun, Baidu, Tencent Cloud, Alibaba Cloud, Google, DeepL): bukan nilai dijanjikan atau diukur sendiri, rujukan sahaja; kuota dan QPS berubah ikut dasar rasmi:

Asas anggaran: sumber rujukan + ekstrapolasi masa (dok rasmi 2026 Youdao / Baidu / Tencent / Alibaba / Google / DeepL); berubah ikut dasar dan versi; ikut konsol rasmi terkini.

Diagnosis pantas mengikut senario sebenar

  • Berfungsi di setempat, tetapi banyak 206 di awan/bekas: jam bekas hanyut dan membatalkan curtime (tandatangan ~120s) → aktifkan NTP, segerakkan jam dengan hos; jangan cache curtime lama.
  • Inggeris terjemah baik, Cina panjang kadang 202: truncation memotong multibait mengikut «bait» → input mesti kira aksara Unicode (q>20: 10 + panjang + 10).
  • salt tetap atau menaik dan 207 pada panggilan kedua: salt mesti rawak (UUID) setiap kali dan segarkan curtime; semasa retry, tandatangani semula.
  • Lonjakan serentak semasa ujian beban → banyak 411: QPS asas ~100 dilampaui → token bucket + backoff di klien, atau beli QPS di konsol.
  • App baharu memanggil sebutan dan dapat 110: instance terjemahan diikat, TTS tidak → ikat setiap instance per perkhidmatan di konsol.
  • Tiba-tiba semua 401: pakej aksara / baki habis → tambah nilai dan tetapkan amaran baki; sediakan saluran sandaran pada laluan kritikal.
  • Perangkap sama semasa migrasi dari Baidu: q di-URL-encode sebelum tandatangan → Youdao v3 menandatangani dengan input (peraturan pemotongan), bukan q terkod; tandatangani dahulu, encode q semasa hantar.
