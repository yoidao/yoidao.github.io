Youdao Zhiyun API ایرر کوڈز / ترجمہ API تشخیص

کیا Youdao Zhiyun کے ترجمہ / لغت / OCR / TTS انٹرفیس نے errorCode لوٹایا؟ پہلے تین باتیں واضح کریں: کیا یہ کلائنٹ پر جانچنے کے قابل پیرامیٹر/دستخط کا مسئلہ ہے، یا سرور/بلنگ کا، کیا دستخط اسٹرنگ کی ترتیب اور انکوڈنگ درست ہے، اور کیا QPS سے تجاوز ہوا۔ ایرر کوڈ یا کلیدی لفظ درج کریں (مثلاً 202، 411، 108، دستخط، حد، ٹائم اسٹیمپ): پہلے L1 سرکاری معنیٰ، پھر L2 بنیادی وجہ اور تشخیص، آخر میں حریف موازنہ اور حقیقی واقعات۔ سرکاری دستاویز Youdao Zhiyun API انضمام گائیڈ / ایرر کوڈز پر۔

مرحلہ 1: کوڈ رینج کے مطابق درجہ بندی کریں (L1 - پہلے کس کی غلطی)

Youdao Zhiyun کا errorCode تقریباً چار رینجوں میں بٹتا ہے؛ اندھا دھند آزمانے سے پہلے مقام طے کریں: ① 1xx پیرامیٹر (101 غائب / 102 زبان / 103 طویل / 113 q خالی / 116 strict) - کلائنٹ لازمی اور اقدار جانچے؛ ② 2xx توثیق اور دستخط (108 ID / 202 دستخط / 203 IP / 205 پلیٹ فارم / 206 ٹائم اسٹیمپ / 207 ری پلے) - دستخط اسٹرنگ اور اسناد جانچیں؛ ③ 3xx/5xx سرور (303 / 500 / 902000) - request_id کے ساتھ دوبارہ یا ٹکٹ؛ ④ 4xx اکاؤنٹ/بلنگ/حد (401 بیلنس / 411 / 412) - بیلنس اور QPS جانچیں۔ مزید v3 دستخط سازی اور 202 تشخیص پر۔

مرحلہ 2: سب سے عام دو جال → بنیادی وجہ (L2: دستخط 202 اور حد 411)

اہم: Youdao API کی 90% خرابیاں «دستخط 202» اور «حد 411» پر اٹکتی ہیں۔ 202 شاذ و نادر ہی غلط کلید ہوتی ہے؛ عموماً غلط بنی v3 دستخط اسٹرنگ - sign = sha256(appKey + input + salt + curtime + appSecret)، جہاں q>20 پر input «پہلے 10 حروف + q کی لمبائی + آخری 10 حروف» لیتا ہے (بائٹ نہیں، Unicode حرف کے حساب سے)، اور ≤20 پر q۔ اس میں UTF-8 انکوڈنگ، ترتیب اور خالی جگہیں شامل ہوتی ہیں۔ 411 پلان سے زیادہ QPS ہے: کلائنٹ پر ریٹ محدود کریں + backoff۔ نیچے خود جانچ پینل، پھر موازنہ اور 2026 اندازے۔

پلیٹ فارمز کی سرکاری دستاویزات (Youdao Zhiyun، Baidu، Tencent Cloud، Alibaba Cloud، Google، DeepL) سے 2026 اندازہ: وعدہ شدہ یا براہِ راست ناپے گئے اقدار نہیں، صرف حوالہ؛ کوٹہ اور QPS سرکاری پالیسی سے بدلتے ہیں:

حقیقی منظر کے مطابق فوری تشخیص

  • لوکل پر چلتا ہے، مگر کلاؤڈ/کنٹینر پر بہت 206: کنٹینر گھڑی بہک کر curtime کو ناکارہ کرتی ہے (دستخط ~120s) → NTP فعال کریں، گھڑی کو ہوسٹ سے ہم وقت کریں؛ پرانا curtime کیش نہ کریں۔
  • انگریزی اچھا ترجمہ، طویل چینی کبھی 202: truncation نے ملٹی بائٹ کو «بائٹ» سے کاٹا → input کو Unicode حروف گننے چاہئیں (q>20: 10 + لمبائی + 10)۔
  • مستقل یا بڑھتا salt اور دوسری کال پر 207: salt ہر بار بے ترتیب (UUID) ہو اور curtime تازہ کریں؛ retry پر دوبارہ دستخط کریں۔
  • لوڈ ٹیسٹ میں ہم وقت چوٹی → بہت 411: بنیادی QPS ~100 سے تجاوز → کلائنٹ پر token bucket + backoff، یا کنسول سے QPS خریدیں۔
  • نئی ایپ تلفظ کال کر کے 110 پاتی ہے: ترجمہ انسٹینس منسلک، TTS نہیں → کنسول میں فی سروس ہر انسٹینس باندھیں۔
  • اچانک سب 401: کریکٹر پیک / بیلنس ختم → ری چارج کریں اور بیلنس الرٹ لگائیں؛ اہم راستوں پر بیک اپ چینل رکھیں۔
  • Baidu سے منتقلی میں وہی جال: دستخط سے پہلے q کو URL-encode کیا → Youdao v3 input سے دستخط کرتا ہے (کٹائی قاعدہ)، انکوڈ شدہ q سے نہیں؛ پہلے دستخط، بھیجتے وقت q انکوڈ۔
