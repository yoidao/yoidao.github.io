Kody błędów API Youdao Zhiyun / diagnoza API tłumaczeń

Interfejs tłumaczenia / słownika / OCR / TTS Youdao Zhiyun zwrócił errorCode? Najpierw wyjaśnij trzy rzeczy: czy to problem parametru/podpisu sprawdzalny po stronie klienta, czy serwera/rozliczeń, czy kolejność i kodowanie ciągu podpisu są poprawne oraz czy przekroczono QPS. Wpisz kod błędu lub słowo kluczowe (np. 202, 411, 108, podpis, limit, znacznik czasu): najpierw oficjalne znaczenie L1, potem przyczyna źródłowa i diagnoza L2, na końcu porównanie z konkurencją i realne przypadki. Oficjalna dok. na przewodniku integracji / kodów błędów Youdao Zhiyun.

Krok 1: sklasyfikuj według zakresu kodu (L1 - najpierw czyja wina)

errorCode Youdao Zhiyun dzieli się z grubsza na cztery zakresy; umiejsców najpierw, zamiast próbować na ślepo: ① 1xx parametry (101 brak / 102 język / 103 długi / 113 q puste / 116 strict) - klient sprawdza wymagane i wartości; ② 2xx auth i podpis (108 ID / 202 podpis / 203 IP / 205 platforma / 206 znacznik czasu / 207 powtórka) - sprawdź ciąg podpisu i poświadczenia; ③ 3xx/5xx serwer (303 / 500 / 902000) - ponów z request_id lub zgłoszenie; ④ 4xx konto/rozliczenia/limit (401 saldo / 411 / 412) - sprawdź saldo i QPS. Więcej na generowaniu podpisu v3 i diagnozie 202.

Parametry101 · 102 · 103 · 113 · 116
Kontrola klienta: pola wymagane kompletne (q/from/to/appKey/sign/signType/curtime/salt), kody języka zh-CHS / en / auto, q niepuste i niezbyt długie, strict true/false.
Auth / podpis108 · 111 · 202 · 203 · 205 · 206 · 207
SHA256 z appKey+input+salt+curtime+appSecret; curtime UTC w sekundach, salt UUID przeciw powtórkom; biała lista IP zgodna z typem platformy.
Usługa / instancja110 · 310
Każda aplikacja wiąże instancję per usługa (tłumaczenie / TTS / OCR osobno); tłumaczenie domenowe aktywuje się wcześniej w konsoli.
Konto / rozliczenia / limit401 · 411 · 412
Nie wyczerpuj salda / pakietu znaków (401 zatrzymuje usługę); QPS (bazowy ~100) przekroczony = 411: token bucket + backoff wykładniczy.
Wynik / deszyfracja / inne201 · 303 · 500 · 902000
Błędy serwera: ponów z request_id lub zgłoszenie; w interfejsie szyfrowanym sprawdź kolejność DES / BASE64 / URLDecode.

Krok 2: dwie najczęstsze pułapki → przyczyna źródłowa (L2: podpis 202 i limit 411)

Sedno: 90% błędów API Youdao blokuje się na «podpisie 202» i «limicie 411». 202 rzadko to zły klucz, zwykle źle zbudowany ciąg podpisu v3 - sign = sha256(appKey + input + salt + curtime + appSecret), gdzie input przy q>20 bierze «10 pierwszych znaków + długość q + 10 ostatnich znaków» (wg znaków Unicode, nie bajtów), a q przy ≤20. Dodaj kodowanie UTF-8, kolejność i spacje. 411 to QPS powyżej planu: ogranicz tempo + backoff po stronie klienta. Poniżej panel autokontroli, potem porównanie i szacunki 2026.

Szybka diagnoza wg realnego scenariusza

  • Działa lokalnie, ale dużo 206 w chmurze/kontenerze: zegar kontenera dryfuje i unieważnia curtime (podpis ~120s) → włącz NTP, zsynchronizuj zegar z hostem; nie buforuj starych curtime.
  • Angielski tłumaczy dobrze, długi chiński czasem 202: obcinanie pocięło wielobajt wg «bajtów» → input musi liczyć znaki Unicode (q>20: 10 + długość + 10).
  • stały lub rosnący salt i 207 przy drugim wywołaniu: salt musi być losowy (UUID) za każdym razem i odświeżać curtime; przy ponowieniu podpisz ponownie.
  • Skoki współbieżności w teście obciążeniowym → dużo 411: bazowy QPS ~100 przekroczony → token bucket + backoff po stronie klienta, lub dokup QPS w konsoli.
  • Nowa aplikacja wywołuje wymowę i dostaje 110: instancja tłumaczenia powiązana, TTS nie → powiąż każdą instancję per usługa w konsoli.
  • Nagle wszystko 401: pakiet znaków / saldo wyczerpane → doładuj i ustaw alerty salda; miej kanał zapasowy na ścieżkach krytycznych.
  • Ta sama pułapka przy migracji z Baidu: q zakodowane URL przed podpisem → Youdao v3 podpisuje z input (reguła obcinania), nie z zakodowanym q; najpierw podpisz, zakoduj q przy wysyłce.
