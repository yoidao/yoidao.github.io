Kode error API Youdao Zhiyun / diagnosis API terjemahan

Antarmuka terjemahan / kamus / OCR / TTS Youdao Zhiyun mengembalikan errorCode? Perjelas tiga hal dulu: apakah masalah parameter/tanda tangan yang bisa dicek klien, atau server/penagihan, apakah urutan dan encoding string tanda tangan benar, dan apakah QPS terlampaui. Masukkan kode error atau kata kunci (mis. 202, 411, 108, tanda tangan, batas, timestamp): pertama arti resmi L1, lalu akar masalah dan diagnosis L2, terakhir perbandingan pesaing dan kasus nyata. Dok resmi di panduan integrasi / kode error Youdao Zhiyun.

Langkah 1: golongkan berdasarkan rentang kode (L1 - dulu siapa yang salah)

errorCode Youdao Zhiyun terbagi kira-kira empat rentang; tempatkan dulu sebelum coba-coba: ① 1xx parameter (101 hilang / 102 bahasa / 103 panjang / 113 q kosong / 116 strict) - klien cek wajib dan nilai; ② 2xx auth & tanda tangan (108 ID / 202 tanda tangan / 203 IP / 205 platform / 206 timestamp / 207 replay) - cek string tanda tangan dan kredensial; ③ 3xx/5xx server (303 / 500 / 902000) - coba lagi dengan request_id atau tiket; ④ 4xx akun/penagihan/batas (401 saldo / 411 / 412) - cek saldo dan QPS. Lebih lanjut di pembuatan tanda tangan v3 dan diagnosis 202.

Parameter101 · 102 · 103 · 113 · 116
Cek klien: field wajib lengkap (q/from/to/appKey/sign/signType/curtime/salt), kode bahasa zh-CHS / en / auto, q tidak kosong/terlalu panjang, strict true/false.
Auth / tanda tangan108 · 111 · 202 · 203 · 205 · 206 · 207
SHA256 dari appKey+input+salt+curtime+appSecret; curtime UTC detik, salt UUID anti-replay; whitelist IP selaras tipe platform.
Layanan / instance110 · 310
Tiap app mengikat satu instance per layanan (terjemahan / TTS / OCR terpisah); terjemahan domain diaktifkan dulu di konsol.
Akun / penagihan / batas401 · 411 · 412
Jangan habiskan saldo / paket karakter (401 menghentikan layanan); QPS (basis ~100) terlampaui = 411: token bucket + backoff eksponensial.
Hasil / dekripsi / lainnya201 · 303 · 500 · 902000
Error server: coba lagi dengan request_id atau tiket; pada antarmuka terenkripsi cek urutan DES / BASE64 / URLDecode.

Langkah 2: dua jebakan tersering → akar masalah (L2: tanda tangan 202 dan batas 411)

Inti: 90% error API Youdao tersangkut di «tanda tangan 202» dan «batas 411». 202 jarang karena kunci salah, biasanya string tanda tangan v3 salah susun - sign = sha256(appKey + input + salt + curtime + appSecret), di mana input saat q>20 ambil «10 karakter pertama + panjang q + 10 karakter terakhir» (per karakter Unicode, bukan byte), dan q jika ≤20. Tambah encoding UTF-8, urutan, dan spasi. 411 adalah QPS di atas paket: batasi laju + backoff di klien. Di bawah panel swacek, lalu perbandingan dan estimasi 2026.

Diagnosis cepat per skenario nyata

  • Jalan di lokal, tapi banyak 206 di cloud/kontainer: jam kontainer melenceng dan membatalkan curtime (tanda tangan ~120s) → aktifkan NTP, sinkronkan jam dengan host; jangan cache curtime lama.
  • Inggris terjemah baik, Mandarin panjang kadang 202: truncation memotong multibyte per «byte» → input harus hitung karakter Unicode (q>20: 10 + panjang + 10).
  • salt tetap atau menaik dan 207 di panggilan kedua: salt harus acak (UUID) tiap kali dan segarkan curtime; saat retry, tanda tangani ulang.
  • Lonjakan konkurensi saat uji beban → banyak 411: QPS basis ~100 terlampaui → token bucket + backoff di klien, atau beli QPS di konsol.
  • App baru memanggil pelafalan dan dapat 110: instance terjemahan terikat, TTS tidak → ikat tiap instance per layanan di konsol.
  • Tiba-tiba semua 401: paket karakter / saldo habis → isi ulang dan atur peringatan saldo; siapkan kanal cadangan di jalur kritis.
  • Jebakan sama saat migrasi dari Baidu: q di-URL-encode sebelum tanda tangan → Youdao v3 menandatangani dengan input (aturan pemotongan), bukan q ter-encode; tandatangani dulu, encode q saat kirim.
