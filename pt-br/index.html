Códigos de erro da API Youdao Zhiyun / diagnóstico da API de tradução

A interface de tradução / dicionário / OCR / TTS da Youdao Zhiyun retornou um errorCode? Esclareça três coisas primeiro: é problema de parâmetro/assinatura que o cliente pode checar, ou de servidor/cobrança, a ordem e a codificação da string de assinatura estão certas e o QPS foi excedido. Digite o código de erro ou palavra-chave (ex.: 202, 411, 108, assinatura, limite, timestamp): primeiro o significado oficial L1, depois a causa raiz e o diagnóstico L2, por fim a comparação com concorrentes e casos reais. Doc oficial em o guia de integração / códigos de erro da Youdao Zhiyun.

Passo 1: classifique pela faixa de código (L1 - primeiro de quem é a culpa)

O errorCode da Youdao Zhiyun se divide em quatro faixas; localize antes de tentar às cegas: ① 1xx parâmetros (101 ausente / 102 idioma / 103 longo / 113 q vazio / 116 strict) - o cliente confere obrigatórios e valores; ② 2xx auth e assinatura (108 ID / 202 assinatura / 203 IP / 205 plataforma / 206 timestamp / 207 replay) - confira a string de assinatura e credenciais; ③ 3xx/5xx servidor (303 / 500 / 902000) - tente de novo com request_id ou abra ticket; ④ 4xx conta/cobrança/limite (401 saldo / 411 / 412) - confira saldo e QPS. Mais detalhes em geração de assinatura v3 e diagnóstico do 202.

Parâmetros101 · 102 · 103 · 113 · 116
Verificação no cliente: campos obrigatórios completos (q/from/to/appKey/sign/signType/curtime/salt), códigos de idioma zh-CHS / en / auto, q não vazio nem longo demais, strict em true/false.
Auth / assinatura108 · 111 · 202 · 203 · 205 · 206 · 207
Assinatura SHA256 de appKey+input+salt+curtime+appSecret; curtime UTC em segundos, salt UUID anti-replay; whitelist de IP alinhada ao tipo de plataforma.
Serviço / instância110 · 310
Cada app vincula uma instância por serviço (tradução / TTS / OCR separados); a tradução por domínio é ativada antes no console.
Conta / cobrança / limite401 · 411 · 412
Não esgote saldo / pacote de caracteres (401 corta o serviço); QPS (base ~100) excedido = 411: token bucket + backoff exponencial.
Resultado / descriptografia / outros201 · 303 · 500 · 902000
Erros de servidor: tente de novo com request_id ou ticket; em interface criptografada, confira a ordem DES / BASE64 / URLDecode.

Passo 2: as duas armadilhas mais frequentes → causa raiz (L2: assinatura 202 e limite 411)

Chave: 90% dos erros da API Youdao travam em «assinatura 202» e «limite 411». O 202 raramente é chave errada, mas costuma ser uma string de assinatura v3 mal montada - sign = sha256(appKey + input + salt + curtime + appSecret), em que input com q>20 pega «10 primeiros caracteres + tamanho de q + 10 últimos caracteres» (por caracteres Unicode, não bytes), e q se ≤20. Some codificação UTF-8, ordem e espaços. O 411 é QPS acima do plano: limite a taxa + backoff no cliente. Abaixo, o painel de autoverificação e depois a comparação e as estimativas 2026.

Diagnóstico rápido por cenário real

  • Funciona no local, mas muitos 206 na nuvem/contêiner: o relógio do contêiner desvia e invalida o curtime (assinatura ~120s) → ative NTP, sincronize o relógio com o host; não faça cache de curtime antigo.
  • Inglês traduz bem, mas chinês longo dá 202 às vezes: o truncamento cortou o multibyte por «bytes» → input deve contar caracteres Unicode (q>20: 10 + tamanho + 10).
  • salt fixo ou autoincremental e um 207 na segunda chamada: o salt deve ser aleatório (UUID) a cada vez e renovar o curtime; no retry, reassine.
  • Picos de concorrência em teste de carga → muitos 411: QPS base ~100 excedido → token bucket + backoff no cliente, ou compre QPS no console.
  • App novo chamando a pronúncia e recebendo 110: a instância de tradução está vinculada, a de TTS não → vincule cada instância por serviço no console.
  • De repente tudo em 401: pacote de caracteres / saldo esgotado → recarregue e configure alertas de saldo; tenha um canal reserva em caminhos críticos.
  • Mesma armadilha ao migrar do Baidu: q codificado em URL antes de assinar → Youdao v3 assina com input (regra de truncamento), não com q codificado; assine primeiro, codifique q no envio.
