Youdao Zhiyun API ত্রুটি কোড / অনুবাদ API নির্ণয়

Youdao Zhiyun-এর অনুবাদ / অভিধান / OCR / TTS ইন্টারফেস errorCode ফেরত দিয়েছে? প্রথমে তিনটি বিষয় স্পষ্ট করুন: এটি ক্লায়েন্টে যাচাইযোগ্য প্যারামিটার/স্বাক্ষরের সমস্যা, নাকি সার্ভার/বিলিং, স্বাক্ষর স্ট্রিংয়ের ক্রম ও এনকোডিং ঠিক কি না, এবং QPS ছাড়িয়ে গেছে কি না। ত্রুটি কোড বা কীওয়ার্ড দিন (যেমন 202, 411, 108, স্বাক্ষর, সীমা, টাইমস্ট্যাম্প): প্রথমে L1 অফিসিয়াল অর্থ, তারপর L2 মূল কারণ ও নির্ণয়, শেষে প্রতিযোগী তুলনা ও বাস্তব ঘটনা। অফিসিয়াল ডক Youdao Zhiyun API ইন্টিগ্রেশন গাইড / ত্রুটি কোড-এ।

ধাপ ১: কোড পরিসীমা অনুযায়ী শ্রেণিবদ্ধ করুন (L1 - প্রথমে কার দোষ)

Youdao Zhiyun-এর errorCode মোটামুটি চারটি পরিসরে ভাগ হয়; অন্ধভাবে চেষ্টার আগে অবস্থান নির্ণয় করুন: ① 1xx প্যারামিটার (101 নেই / 102 ভাষা / 103 দীর্ঘ / 113 q খালি / 116 strict) - ক্লায়েন্ট আবশ্যক ও মান যাচাই করে; ② 2xx প্রমাণীকরণ ও স্বাক্ষর (108 ID / 202 স্বাক্ষর / 203 IP / 205 প্ল্যাটফর্ম / 206 টাইমস্ট্যাম্প / 207 রিপ্লে) - স্বাক্ষর স্ট্রিং ও শংসাপত্র যাচাই করুন; ③ 3xx/5xx সার্ভার (303 / 500 / 902000) - request_id দিয়ে পুনরায় বা টিকিট; ④ 4xx অ্যাকাউন্ট/বিলিং/সীমা (401 ব্যালেন্স / 411 / 412) - ব্যালেন্স ও QPS যাচাই করুন। আরও v3 স্বাক্ষর তৈরি ও 202 নির্ণয়-এ।

ধাপ ২: সবচেয়ে সাধারণ দুই ফাঁদ → মূল কারণ (L2: স্বাক্ষর 202 ও সীমা 411)

মূল কথা: Youdao API-এর 90% ত্রুটি «স্বাক্ষর 202» ও «সীমা 411»-এ আটকে যায়। 202 খুব কমই ভুল কী; সাধারণত ভুলভাবে গঠিত v3 স্বাক্ষর স্ট্রিং - sign = sha256(appKey + input + salt + curtime + appSecret), যেখানে q>20 হলে input «প্রথম 10 অক্ষর + q-এর দৈর্ঘ্য + শেষ 10 অক্ষর» নেয় (বাইট নয়, Unicode অক্ষর অনুযায়ী), আর ≤20 হলে q। সাথে UTF-8 এনকোডিং, ক্রম ও স্পেস। 411 হলো প্ল্যানের বেশি QPS: ক্লায়েন্টে রেট সীমিত করুন + ব্যাকঅফ। নিচে স্ব-যাচাই প্যানেল, তারপর তুলনা ও 2026 অনুমান।

বাস্তব দৃশ্যপট অনুযায়ী দ্রুত নির্ণয়

  • লোকালে চলে, কিন্তু ক্লাউড/কন্টেইনারে প্রচুর 206: কন্টেইনার ঘড়ি সরে গিয়ে curtime অকার্যকর করে (স্বাক্ষর ~120s) → NTP চালু করুন, ঘড়ি হোস্টের সাথে সিঙ্ক করুন; পুরনো curtime ক্যাশ করবেন না।
  • ইংরেজি ভালো অনুবাদ, দীর্ঘ চীনা মাঝে মাঝে 202: truncation মাল্টিবাইট «বাইট» অনুযায়ী কেটেছে → input-কে Unicode অক্ষর গুনতে হবে (q>20: 10 + দৈর্ঘ্য + 10)।
  • স্থির বা বর্ধমান salt এবং দ্বিতীয় কলে 207: salt প্রতিবার এলোমেলো (UUID) হতে হবে এবং curtime রিফ্রেশ; retry-তে আবার স্বাক্ষর করুন।
  • লোড টেস্টে সমকালীন শিখর → প্রচুর 411: বেস QPS ~100 অতিক্রম → ক্লায়েন্টে token bucket + ব্যাকঅফ, বা কনসোল থেকে QPS কিনুন।
  • নতুন অ্যাপ উচ্চারণ কল করে 110 পায়: অনুবাদ ইনস্ট্যান্স বাঁধা, TTS নয় → কনসোলে প্রতি সেবায় প্রতিটি ইনস্ট্যান্স বাঁধুন।
  • হঠাৎ সব 401: অক্ষর প্যাক / ব্যালেন্স শেষ → রিচার্জ করুন ও ব্যালেন্স অ্যালার্ট দিন; গুরুত্বপূর্ণ পথে ব্যাকআপ চ্যানেল রাখুন।
  • Baidu থেকে মাইগ্রেশনে একই ফাঁদ: স্বাক্ষরের আগে q-কে URL-encode করা → Youdao v3 input দিয়ে স্বাক্ষর করে (কাটার নিয়ম), এনকোডেড q দিয়ে নয়; আগে স্বাক্ষর, পাঠানোর সময় q এনকোড।
