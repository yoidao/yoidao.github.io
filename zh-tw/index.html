有道智雲 API 錯誤碼 / 翻譯介面報錯排查查詢

呼叫有道智雲翻譯 / 詞典 / OCR / TTS 介面回傳了 errorCode？先想清三件事——是用戶端可自查的參數 / 簽章問題，還是伺服端 / 計費問題、簽章字串順序與編碼對不對、QPS 有沒有超。輸入或點選你拿到的錯誤碼或關鍵字（如 202、411、108、簽章、限流、時間戳），先看 L1 官方含義，再看 L2 深度真因與排查，最後對照競品與真實情境。需要官方接入文件可到 有道智雲 API 錯誤碼 / 接入指南。

第一步：按錯誤碼區間分流（L1·先分清誰的鍋）

有道智雲的 errorCode 大致分四段，先按段定位、別瞎試：① 1xx 參數類（101 缺參 / 102 語種 / 103 過長 / 113 q 空 / 116 strict）——用戶端自查必填與取值；② 2xx 鑑權與簽章類（108 應用ID / 202 簽章 / 203 IP / 205 平台 / 206 時間戳 / 207 重放）——查簽章字串與憑證；③ 3xx/5xx 伺服端類（303 / 500 / 902000）——帶 request_id 重試或開工單；④ 4xx 帳號/計費/限流類（401 欠費 / 411 / 412 限流）——查餘額與 QPS。更完整的接入對照見 有道 API 簽章(v3)產生與 202 排查。

參數類101 · 102 · 103 · 113 · 116
用戶端自查：必填欄位齊全（q/from/to/appKey/sign/signType/curtime/salt），語種代碼用 zh-CHS / en / auto，q 非空且不超長，strict 取 true/false。
鑑權 / 簽章類108 · 111 · 202 · 203 · 205 · 206 · 207
簽章字串 appKey+input+salt+curtime+appSecret 做 SHA256；curtime 秒級 UTC、salt 用 UUID 防重放；IP 白名單與應用平台類型對齊。
服務 / 實例類110 · 310
一個應用要依服務逐個綁定實例（翻譯 / 語音合成 TTS / OCR 各自獨立）；領域化翻譯需先在控制台開通。
帳號 / 計費 / 限流類401 · 411 · 412
餘額 / 字元資源包別用盡（401 欠費停服）；QPS（基礎約 100）超限會 411，用戶端做權杖桶限速 + 指數退避。
結果 / 解密 / 其它類201 · 303 · 500 · 902000
伺服端類異常帶 request_id 重試或開工單；加密介面核對 DES / BASE64 / URLDecode 順序。

第二步：最高頻兩個坑 → 深度真因（L2·簽章 202 與限流 411）

關鍵認知：90% 的有道 API 報錯卡在「簽章 202」和「限流 411」上。202 不是金鑰錯，多半是 v3 簽章字串沒拼對——sign = sha256(應用ID + input + salt + curtime + 應用金鑰)，其中 input 在 q 長度>20 時取「前 10 字元 + q 長度 + 後 10 字元」（按 Unicode 字元、非位元組），≤20 取 q 本身；再疊加 UTF-8 編碼、順序、首尾空白幾個坑。411 則是 QPS 超了方案上限，要用戶端限速 + 退避。下圖是呼叫前自檢面板示意；再往下是競品橫向對照與 2026 估算。

真實情境速判

  • 本機能調、上雲 / 容器後大面積 206：容器時鐘漂移導致 curtime 失效（簽章約 120s 有效）→ 開 NTP 校時、容器同步宿主時鐘，別快取舊 curtime。
  • 翻譯英文正常、翻譯長中文偶發 202：truncate 把多位元組中文按「位元組」切了 → input 必須按 Unicode 字元數（q>20 取前 10 字元 + q 長度 + 後 10 字元）。
  • salt 寫死或自增、第二次呼叫就 207：salt 必須每次隨機（UUID）並刷新 curtime，失敗重試要重新簽章、不能原樣重送。
  • 壓測瞬時並發大面積 411：基礎 QPS 約 100，超了被限流 → 用戶端權杖桶限速 + 指數退避，或控制台疊加購買 QPS。
  • 新建應用直接取翻譯發音報 110：翻譯實例綁了、語音合成（TTS）實例沒綁 → 控制台為應用依服務逐個綁定實例。
  • 呼叫突然全部 401：字元資源包 / 餘額用盡欠費停服 → 儲值或購買資源包並設餘額告警，關鍵鏈路備兜底通道。
  • 從百度遷來的同坑：簽章拼接前就對 q 做了 URL encode → 有道 v3 用 input（截斷規則）參與簽章、不是先對 q 編碼；應先簽章、發請求時再對 q 做 URL encode。
